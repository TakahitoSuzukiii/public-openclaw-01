## 0. 前提：Claude 各モデルの概要（簡単な要約）

Anthropic の Claude は、用途・コスト・知能のバランスで複数のモデルが並行提供されています。2026年6月時点の「現行（推奨）」モデルは以下のとおりです。

| モデル | モデル ID | コンテキスト窓 | 出力上限 | 入力 $/MTok | 出力 $/MTok | 位置づけ |
|---|---|---|---|---|---|---|
| **Claude Fable 5** | `claude-fable-5` | 1M | 128K | $10.00 | $50.00 | 最上位・最高知能（Opus 上位の新ティア） |
| **Claude Opus 4.8** | `claude-opus-4-8` | 1M | 128K | $5.00 | $25.00 | Opus ティア最上位・高自律エージェント向け |
| Claude Opus 4.7 | `claude-opus-4-7` | 1M | 128K | $5.00 | $25.00 | 前世代 Opus |

ざっくりの使い分け:
- **Haiku** … 分類・要約など軽量＆速度重視。

- **Fable** … Opus でも足りない「最難関」タスク向けの最上位（コストは Opus の 2倍）。

## 1. Fable 5 とは（調査本題）

**Claude Fable 5 は、現行で最も高性能・高知能のフラッグシップモデル**です。Opus の「ひとつ上」に新設された最上位ティアの一般提供モデルとして追加されました（一般提供の正確な開始日は公式リファレンスでは未確認）。

### 主なスペック
- モデル ID: `claude-fable-5`
- コンテキスト窓: **1M トークン**（長文コンテキストの追加料金なし）
- 出力トークン上限: **128K**（大きい出力はストリーミング必須）
- 料金: **入力 $10 / 出力 $50（per MTok）** = Opus 4.8 の **約2倍**
- 機能: ビジョン（画像入力）、ツール使用／関数呼び出し、構造化出力、アダプティブ思考、effort パラメータ（`xhigh`・`max` 含む）、プロンプトキャッシュ、コンパクション（beta）に対応。

### 立ち位置
知能・コーディング・エージェント系ベンチマークで現行トップクラス。「Opus 4.8 でも難しい、正確性が最優先で多少のコスト増を許容できる」ケースで選ぶモデル、という整理になります。日常運用の既定モデルとして常用するには割高です。

## 2. Fable 5 と Opus 4.8・他モデルとの違い

### (A) コスト
- Fable 5 = **Opus 4.8 の 2倍**（入力 $10 vs $5、出力 $50 vs $25）。
- Opus 4.8 自体は Opus 4.6/4.7 と同額（$5/$25）。Sonnet 4.6 は $3/$15、Haiku 4.5 は $1/$5。
- → 「Fable は明確に高価」。常用は Opus/Sonnet、ここぞの場面で Fable、が基本戦略。

### (B) 知能・能力
- Fable 5 が最上位。Opus 4.8 は Opus ティアの最上位で、長時間の自律エージェント作業・ナレッジワーク・メモリ系が非常に強い。
- 一般的なタスクでは Opus 4.8 で十分なことが多く、Fable は「最難関タスクでの上振れ」を狙う位置づけ。

### (C) API の仕様差（重要・実装に影響）
Fable 5 は Opus 4.7/4.8 とほぼ同じリクエスト仕様を継承しますが、**Fable 固有の破壊的変更が1点**あります。

| 項目 | Fable 5 | Opus 4.8 |
|---|---|---|
| `thinking: {type:"enabled", budget_tokens:N}` | **400 エラー**（削除済み） | 同じく 400 エラー |
| `temperature` / `top_p` / `top_k` | **400 エラー**（削除済み） | 同じく 400 エラー |
| 思考の指定方法 | `thinking: {type:"adaptive"}` を使う | 同じ |
| `thinking: {type:"disabled"}`（明示的に無効化） | **400 エラー** → `thinking` パラメータ自体を省略する | **許容される**（4.7/4.8 では OK） |
| プロンプトキャッシュ最小プレフィックス | **2048 トークン** | 4096 トークン |
| effort パラメータ（`low`〜`max`、`xhigh`） | 対応 | 対応 |
| アシスタント・プレフィル（最後の assistant ターンで続きを書かせる） | **400 エラー**（構造化出力 or システムプロンプトで代替） | 同じく 400 エラー |

実装上の要点（Fable を使う場合）:
- 思考を**使う**: `thinking: {type:"adaptive"}` を指定。
- 思考を**使わない**: `thinking` パラメータを**書かない**（`disabled` を渡すと 400）。Opus 4.8 とここだけ挙動が違う。
- `temperature` 等のサンプリングパラメータは渡さない（渡すと 400）。挙動の制御はプロンプトで行う。
- 出力が大きい場合（〜128K）は**ストリーミング必須**。

### (D) 他モデルとの簡易比較
- vs Sonnet 4.6 … Sonnet は安く速いが知能上限は下。出力上限 64K。
- vs Haiku 4.5 … Haiku は最安・最速だがコンテキスト 200K と小さく、簡易タスク向け。

## 3. このOpenClawサーバで使えるのか

**結論: プロバイダ（anthropic）経由で技術的には到達可能。ただし「既定エイリアス未設定」かつ「APIアカウント側の Fable 利用権限が前提」。実利用の確認には別途テスト（外部API課金）が必要。**

調査時点の本サーバ構成（ランタイム情報より）:
- 既定モデル: `anthropic/claude-opus-4-8`
- 定義済みエイリアス: `opus` → `anthropic/claude-opus-4-8`、`sonnet` → `anthropic/claude-sonnet-4-6`
- → **`fable` というエイリアスは未定義**。Fable を使う場合はフルの provider/model 文字列 `anthropic/claude-fable-5` を指定する形になる。

判断の根拠と留意点:
- 本サーバは anthropic プロバイダを使っており、同プロバイダはモデル文字列のパススルーが可能。よって `anthropic/claude-fable-5` を指定すれば**ルーティング上は到達できる**見込み。
- ただし Fable は Opus 上位の新ティアであり、**実際に呼べるかは API キーの組織（org）が Fable へのアクセス権を持つかに依存**する（新ティアはアクセス権が分かれている場合がある）。
- コスト面: Opus の 2倍。常用設定にはせず、必要時のみのセッション単位オーバーライド（モデル切替）での利用を推奨。

> 本ポーラ（タスク実行系）は、設定/スケジューラ変更・外部送信・課金を伴う操作を自走で行わないため、**「実際に `claude-fable-5` を呼べるか」のライブ確認は本ドキュメントでは未実施**。確認したい場合は、セッションのモデルオーバーライドで `anthropic/claude-fable-5` を指定し、短い疎通テスト（少量トークン）を鈴木さんの承認のもとで実施するのが安全。

推奨アクション（任意）:
1. まず Opus 4.8 を既定のまま運用（コスト最適）。
2. Fable を試すなら、`fable` エイリアス（`anthropic/claude-fable-5`）を config に追加し、最小トークンで疎通テスト。
3. アクセス権エラー（403/permission）が出る場合は、Anthropic 側でアカウントの Fable 利用可否を確認。
