# AI関連の専門用語・語源・類語の整理

> 作成日 2026-06-16 ／ カテゴリ: AI / 用語整理
> 目的: 各用語の「意味」「単語の元の意味（語源）」「類語・間違えやすい近い言葉の区別」をまとめてキャッチアップする。
> 注: Claude のモデル情報は Anthropic 公式リファレンス（claude-api スキル）で確認。その他は一般に確立した定義に基づく。

## 0. 早見表

| 用語 | 一言で | 最も混同しやすい相手 |
|---|---|---|
| AI-ready data | AIがそのまま使える状態に整えられたデータ | ビッグデータ／構造化データ |
| RAG | 外部知識を検索して生成に渡す手法 | ファインチューニング |
| AIエージェント | 目標に向けて自律的に判断・行動するAI | LLM単体／ワークフロー自動化 |
| AIネイティブ | 最初からAI前提で作られた製品・組織 | AI対応（後付け） |
| Claude Code | Anthropic公式のエージェント型コーディングツール | ChatのClaude／IDE補完 |
| Claude Mythos 5 | Fable 5 と同等のClaude最上位モデル（限定提供） | Fable 5 |
| Fable 5 | Anthropic最高性能の一般提供モデル | Opus 4.x |

## 1. AI-ready data（AIレディ・データ）

**意味:** AI／機械学習がそのまま学習・推論に利用できる状態まで整備されたデータ。具体的には「クリーニング済み・正規化済み・ラベル付け（または構造化）済み・出所と権利が明確・最新で一貫性がある」データを指す。単に量があるだけの生データとは区別される。

**単語の元の意味（語源）:**
- **ready** … 古英語 *ræde*（「準備ができた、すぐ使える」）。「-ready」を後置して「〜にすぐ使える状態」を表す（例: oven-ready=オーブンに入れればすぐ焼ける）。AI-ready は「AIにかければすぐ使える」状態を意味する複合語。
- つまり「データがAIにとって即戦力になっている」というニュアンス。

**類語・区別:**
- **ビッグデータ（big data）** … 量・速度・多様性が大きいデータ。"大きい"ことが主眼で、整っているとは限らない。AI-ready は「大きさ」ではなく「使える品質」が主眼。
- **構造化データ（structured data）** … 表形式など決まった構造を持つデータ。AI-ready の一要素ではあるが、構造化されていても古い・偏っていれば AI-ready ではない。
- **クレンジング済みデータ（cleansed data）** … 誤り・欠損を除いたデータ。AI-ready はクレンジングに加え、ラベル付け・権利・鮮度まで含むより広い概念。

## 2. RAG（Retrieval-Augmented Generation／検索拡張生成）

**意味:** LLM が回答を生成する前に、外部の知識ソース（社内文書・DB・Webなど）を**検索（retrieval）して関連情報を取り出し**、それをプロンプトに添えて**生成（generation）**させる手法。モデル本体を再学習せずに、最新・専門・社内の知識を反映でき、幻覚（hallucination）を抑え、出典を示しやすい。

**単語の元の意味（語源）:**
- **retrieval** … 古フランス語 *retrover*（再び見つける）→ 英語 retrieve。「取り戻す・検索して取り出す」。
- **augmented** … ラテン語 *augere*（増やす・拡大する）。AR（拡張現実）の "augmented" と同語源で「本来のものに足して強化する」。
- **generation** … ラテン語 *generare*（生み出す）。
- 直訳すると「検索で“増強”された生成」。

**類語・区別:**
- **ファインチューニング（fine-tuning）** … モデルの重み自体を追加学習で更新する。RAG は重みを変えず外部知識を“その場で渡す”点が決定的に異なる。最新情報や頻繁に変わる知識は RAG、振る舞い・口調の作り込みはファインチューニング、と使い分ける。
- **プロンプトエンジニアリング** … プロンプトの書き方の工夫。RAG は「検索結果をプロンプトに自動で差し込む仕組み」で、プロンプト工夫の一種だが検索パイプラインを伴う点が違う。
- **グラウンディング（grounding）** … 回答を事実・出典に“接地”させること。RAG はグラウンディングを実現する代表的手段の一つ（RAG=手段、grounding=目的に近い）。

## 3. AIエージェント（AI agent）

**意味:** 与えられた目標に対して、状況を観察し、自分で計画を立て、ツール（検索・コード実行・API呼び出しなど）を使い、結果を見て次の行動を決める——という**自律的なループ**で動く AI システム。単発の質問応答（LLM単体）と違い、「複数ステップを自分で回して目的を達成する」点が核心。

**単語の元の意味（語源）:**
- **agent** … ラテン語 *agere*（行う・動かす）の現在分詞 *agens*（行う者）。「代理人・作用するもの」。"act"（行為）と同語源。
- つまり「（人に代わって）自ら行為するもの」が原義。経済学・哲学でも「主体的に行動する主体」を agent と呼ぶ。

**類語・区別:**
- **LLM単体（chatbot）** … 1問1答。エージェントは LLM を“頭脳”として使いつつ、ツール・記憶・ループを足して自律行動する上位概念。
- **ワークフロー自動化（RPA／決め打ちパイプライン）** … 人が手順を固定して自動化する。エージェントは手順を**自分で決める**点が異なる（固定フロー=ワークフロー、自己決定=エージェント）。
- **マルチエージェント** … 複数のエージェントが分担・協調する構成。AIエージェントの応用形。

## 4. AIネイティブ（AI native）

**意味:** 製品・サービス・組織が、**最初から AI を前提に設計**されていること。後から AI 機能を“付け足した”のではなく、中核の体験や業務プロセス自体が AI を中心に組み立てられている状態を指す。

**単語の元の意味（語源）:**
- **native** … ラテン語 *nativus*（生まれつきの）／*nasci*（生まれる）。「生来の・その土地生まれの」。
- IT では "cloud native"（クラウド前提で生まれた）の用法を踏襲。「生まれた時から AI を母語とする」イメージで、AI が後付けの言語ではなく“母国語”である状態。

**類語・区別:**
- **AI対応／AI-enabled（AI-powered）** … 既存製品に AI 機能を追加した状態。AIネイティブは「土台から AI」、AI対応は「後付け」で対義に近い。
- **デジタルネイティブ** … 生まれた時からデジタル環境にいる世代・組織。語の作り方（〜native）は同じだが対象がデジタル全般。
- **クラウドネイティブ** … 直接の語源的兄弟。AIネイティブはその発想を AI に置き換えたもの。

## 5. Claude Code

**意味:** Anthropic 公式の**エージェント型コーディングツール**（Claude Code）。ターミナル／IDE／デスクトップ等で動き、コードベースを読み・編集し、コマンドを実行し、テストや Git 操作まで多段階で自律的にこなす「エージェントとしての Claude」。単なる補完ではなく、タスクを与えると複数ファイルにまたがって作業を進める。

**単語の元の意味（語源）:**
- **Claude** … Anthropic の LLM ファミリー名（人名 Claude に由来。情報理論の父 Claude Shannon を想起させる命名と解釈されることが多い）。
- **code** … ラテン語 *codex*（書板・法典）。元は「規範・記号の体系」。プログラムの「コード」もこの「記号体系」の意。

**類語・区別:**
- **Chat の Claude（claude.ai）** … 対話アシスタント。Claude Code は同じ Claude を“コーディング作業を自走するエージェント”として使うツールで、ファイル操作・実行権限を持つ点が違う。
- **IDE のコード補完（GitHub Copilot 等の補完）** … 次の数行を提案するのが主。Claude Code は「タスク単位で自律実行」する粒度が大きい。
- **Claude Agent SDK** … Claude Code の基盤を自作アプリに組み込むための開発キット。Claude Code はその SDK 上に立つ完成ツール。

## 6. Claude Mythos 5

**意味:** Anthropic の Claude 最上位クラスのモデル（モデルID `claude-mythos-5`）。**Fable 5 と同じ能力・価格・API挙動**を持つが、提供経路が異なり「Project Glasswing」という限定プログラム参加者のみがアクセスできる。招待制だった旧 Claude Mythos Preview の後継。実質「Fable 5 の限定提供版」と理解してよい（中身は同等、入手経路だけ違う）。

**単語の元の意味（語源）:**
- **Mythos** … ギリシャ語 *mŷthos*（μῦθος、「物語・神話・語られた言葉」）。英語 myth（神話）の語源。「論理（logos）」と対をなす「物語・語り」を指す古典語。
- Fable（寓話）と並べて、いずれも“物語”系の単語をモデル名に採用している点が命名上のテーマ。

**類語・区別:**
- **Fable 5** … 同等の能力・価格の一般提供版。Mythos 5 = 限定経路、Fable 5 = 一般提供、と入手経路で区別する。
- **Claude Mythos Preview** … 招待制だった前身。Mythos 5 はその後継。
- 「mythos5」「mythos 5」などの表記ゆれは同じものを指す。

## 7. Fable 5（Claude Fable 5）

**意味:** Anthropic の**最も高性能な一般提供モデル**（モデルID `claude-fable-5`）。最難関の推論・長時間（long-horizon）のエージェント作業向け。100万トークンのコンテキスト、最大12.8万トークン出力。API挙動が Opus 系と一部異なる（思考は常時オン、新トークナイザで同じ内容でも約30%多くトークン化、安全分類で `refusal` を返しうる、等）。

**単語の元の意味（語源）:**
- **fable** … ラテン語 *fabula*（「話・物語・寓話」）／*fari*（語る）。英語 fable は「教訓を含む短い寓話（イソップ寓話など）」。"fabulous"（途方もない／元は“寓話のような”）と同語源。
- Mythos（神話）と同じく“物語”系の命名テーマに沿う。

**類語・区別:**
- **Opus 4.x（4.8/4.7/4.6）** … Opus 系の最上位。Fable 5 はそれを超える最高性能だが、価格は Opus 帯より高く、API挙動も異なる（上記）。「最高性能が要る時=Fable 5、通常の最新アップグレード先=Opus 4.8」という棲み分け。
- **Sonnet／Haiku** … 速度・コスト重視の下位帯。Fable は性能最優先帯。
- **Mythos 5** … 中身は同等で提供経路だけ違う双子（§6）。

## 8. LangGraph（ラングラフ）

> 依頼の「langragh」は **LangGraph** の表記ゆれ（綴り間違い）と判断。

**意味:** LangChain チームが提供する、**状態（state）を持つ・複数アクターが協調する AI エージェント**を構築するためのフレームワーク。処理の流れを**グラフ（ノード＝処理ステップ、エッジ＝遷移）**として明示的に組み、ループ・分岐・人間の介在（human-in-the-loop）・途中状態の永続化などを制御できる。複雑なエージェントの“制御構造”を堅く書くための基盤。

**単語の元の意味（語源）:**
- **Lang** … LangChain 由来の接頭辞（language=言語、ラテン語 *lingua*「舌・言語」）。
- **graph** … ギリシャ語 *graphē*（γραφή、「書くこと・描いたもの」）／*graphein*（書く・描く）。数学・計算機科学の「グラフ（ノードとエッジの集合）」もこの語。
- 直訳は「言語（モデル）処理をグラフで表す」もの。

**類語・区別:**
- **LangChain** … LLM アプリ開発の汎用ライブラリ（プロンプト・チェーン・ツール連携など）。LangGraph はその上位／姉妹で、**状態とループを持つグラフ構造**に特化。単純な一直線の処理=LangChain のチェーン、複雑な分岐・循環・長時間エージェント=LangGraph、と棲み分ける。
- **エージェントフレームワーク一般（例: 各種 agent ライブラリ）** … LangGraph はその一つで、「グラフで明示的に制御する」点が特色。
- **ステートマシン／ワークフローエンジン** … LangGraph の発想はこれらに近い（状態遷移を図で定義）。AI エージェント向けに最適化したもの、と捉えると分かりやすい。

## 9. まとめ（区別の勘所）

- **データ系**: 「量（ビッグデータ）」と「使える品質（AI-ready）」は別物。
- **知識の入れ方**: 外部知識をその場で渡す＝RAG／重みを更新＝ファインチューニング。
- **自律性**: 固定手順＝ワークフロー／自分で手順を決める＝エージェント。
- **設計思想**: 後付け＝AI対応／土台からAI＝AIネイティブ。
- **Claude**: 対話＝claude.ai／自走コーディング＝Claude Code。性能最優先の一般提供＝Fable 5／同等の限定提供＝Mythos 5。
- **基盤**: 汎用＝LangChain／状態＋グラフでエージェント制御＝LangGraph。
- **命名の小ネタ**: Fable（寓話）・Mythos（神話）はどちらも“物語”系のギリシャ／ラテン由来語で、Anthropic のモデル命名テーマを成している。
